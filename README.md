# m-index-resource

本仓库用于归档和发布多个独立 HTML 页面资源。这些页面以静态 HTML 形式存放，不依赖于特定域名或后端服务，方便直接查看和引用。

## 项目简介

m-index-resource 是一个通用型 HTML 页面集合仓库。主要用途包括：

- 保存和归档临时或长期使用的独立页面
- 提供可直接运行的 HTML 资源索引
- 为各类页面展示需求提供静态文件支持

仓库内容以 HTML 文件为主，不包含复杂的构建流程或动态生成逻辑。

## 目录结构

```
m-index-resource/
├── pages/          # 存放各个独立 HTML 页面
├── assets/         # 页面所需的公共资源（如图片、样式、脚本）
└── index.html      # 可选的主索引文件
```

- `pages/`：每个子目录或文件对应一个独立页面，命名清晰直观。
- `assets/`：存放页面共享的静态资源，按需组织。
- `index.html`：位于根目录的默认索引页，用于列出或导航到各页面。

## 页面归档说明

### 归档方式

- 每个页面以独立的 HTML 文件或文件夹形式存放。
- 页面内引用的资源建议使用相对路径，确保在本地或静态托管环境下均可正常访问。
- 如需引用外部资源（如 CDN 库），请保留来源说明。

## 维护说明

### 添加新页面

1. 在 `pages/` 目录下创建新文件夹或直接放置 HTML 文件。
2. 如有专属资源，建议放置在同级 `assets/` 子目录中。
3. 更新根目录下的索引文件（如有）以包括新条目。

### 更新现有页面

- 直接修改对应 HTML 文件，保持资源引用正确。
- 若涉及公共资源变更，请同步更新 `assets/` 中的文件。

### 删除页面

- 移除对应的文件或文件夹。
- 清理索引文件中的相关条目。

### 注意事项

- 请勿在仓库中存放敏感信息或私密内容。
- 保持页面结构简洁，避免冗余文件。
- 建议为每个页面添加清晰的标题和注释。
